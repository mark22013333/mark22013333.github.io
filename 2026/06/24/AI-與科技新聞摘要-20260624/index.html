AI 與科技新聞摘要 20260624

發表於2026-06-24|更新於2026-06-24|AI與科技新聞
|總字數:2.3k|閱讀時間:7分鐘|瀏覽量:

過去這幾天，幾家 AI 工具不約而同動了同一個地方——不是功能，是帳單。

能力線還是照常往上爬，誰家又快了幾趴、benchmark 又刷新了，這種新聞每週都有。但真正值得停下來看的，是另一條更安靜的線：你用這些工具到底要付多少錢、錢怎麼算、以及這些錢一路往上游流，最後押在了誰身上。把這禮拜的幾條排開來，主角全是「錢」。而錢的事，往往比功能的事更早決定你明年還用不用得起手上這套工具。

一、GitHub Copilot 改成用量計費，用戶在吵「幾個 prompt 就燒光了」

從 6 月 1 日起，GitHub Copilot 全面轉成用量計費（usage-based billing）。每個付費方案改成附一筆月度的 AI Credit 額度——Pro 方案 10 美元、Pro+ 39 美元——超過就按各模型的 token 費率往下扣，連 Copilot 幫你 review code 都要額外吃掉 GitHub Actions 的分鐘數。同時官方準備收掉年費方案，社群討論區一片哀號，不少人反映那點月額度「跑幾個比較重的 prompt 就見底了」。

數字上 Copilot 還是巨無霸——超過 2,600 萬註冊用戶、Fortune 100 裡九成滲透率。但根據 Stack Overflow 的調查，它在專業開發者裡的市佔，一年內從 67% 掉到 51%。

這件事的重點不在貴幾塊錢，在於計費模式換了之後，成本的不確定性整個從供應商轉嫁到你身上了。吃到飽方案的本質，是供應商替你扛住用量波動的風險；改成按 token 計費，等於這份風險原封不動還給使用者。你這個月寫了多少 code、Agent 自己在背景跑了幾輪、哪天心血來潮讓它重構整包專案——以前這些都不影響帳單，現在每一下都在計費。對一個你沒辦法精準預測自己用量的工具，這種轉變很難說是中性的。

原文來源：Changes to GitHub Copilot Plans - GitHub Community

二、Claude 把帳單拆成兩個池子：你「親手用」歸一邊，AI「自己跑」歸另一邊

Anthropic 6 月 15 日起，把 Claude 的計費拆成兩塊。互動式使用——你在網頁、桌面、手機、還有 Claude Code 終端機裡親手敲的那些——繼續算在原本的訂閱裡。但程式化使用就被切出去了：Agent SDK、headless 的 claude -p 旗標、Claude Code 跑在 GitHub Actions 上的那種無人值守任務，全部移到一個獨立的月度 credit pool，額度依方案大小給，超過就按標準 API 費率算。

這條規則畫的那一刀很有意思——它區分的不是「你用了多少」，而是「是你在用，還是 AI 在替你用」。

過去一整年，整個產業都在鼓吹「讓 AI 自己跑、跑久一點、不用你盯」，自主 Agent、背景任務、排程跑批，都往這個方向推。這次拆帳單等於把那條被忽略的成本線攤開來給你看：自動化從來不是免費的，它只是把你「親手點一下」的成本，換成了「Agent 自己跑十輪」的帳單，而後者你更難盯著。倒過來想就清楚了——怎樣最容易在這套新規則下被坑？答案是：寫一個你以為很省事的自動化迴圈，設定條件沒寫死，讓它在背景空轉，月底才發現那個池子早就見底。能自動跑的東西，往往也最容易在你沒看著的時候把錢燒光。

原文來源：AI Coding Tools Pricing Comparison 2026 - Developers Digest

三、Cursor 衝到 20 億美元 ARR，順手把團隊方案漲了一輪

同樣在計費上動刀的還有 Cursor，但它的故事是另一個版本。它的年化營收（ARR）曲線快得不像話：2025 年 1 月 1 億、6 月 5 億、11 月破 10 億，到 2026 年 2 月已經 20 億——從商業化上線到 20 億 ARR 不到兩年，沒有任何一家 B2B SaaS 公司做到過。隨著新版 Composer 2.5 上線，6 月它把團隊方案重新切成 Standard（年繳每席每月 32 美元）和 Premium（每席每月 96 美元）兩級。

把這三家擺在一起看，會發現一個共同動作：免費或吃到飽當誘餌、先把開發者圈進來養成依賴，等你習慣到離不開了，再把計費模式往對供應商有利的方向調。

這是教科書等級的定價策略，沒什麼好意外的。值得在意的是身為使用者該怎麼反應。當一個工具的成長曲線陡到這種程度，它背後一定有極大的變現壓力等著釋放——今天的價格幾乎一定不是最終價格。所以選工具的時候，除了問「現在多少錢、好不好用」，更該先問一句：哪天它漲價、或把我天天用的功能挪進更貴的方案，我搬不搬得走？答得出來，你才是真的在用工具；答不出來，是工具在租用你。

原文來源：AI Coding Agents 2026 Comparison - Lushbinary

四、一份招股文件露了餡：xAI 一個月付 12.5 億美元，向 Anthropic 租算力

錢的事往使用者端收緊，往上游則是另一種量級。SpaceX 向 SEC 遞交的 S-1 文件裡，順帶揭露了幾個數字：xAI 在 2026 年第一季虧損 24 億美元，同期 capex（資本支出）燒掉 77 億；文件裡還寫到一筆交易——租用 300 兆瓦的算力容量，每月 12.5 億美元，一路簽到 2029 年 5 月。

12.5 億美元一個月，是很多上市公司一整年的營收。整個產業現在的真實樣貌，是少數幾家把天文數字的資本，押在「先把算力跟電力鎖死，未來再說」這個賭注上。

這裡藏著一個結構性風險，而且跟前面幾則是同一條線的延伸。當算力與電力這種命脈，被集中綁進極少數幾筆長約裡，整個系統的韌性其實是在下降的——任何一個環節（一家公司的現金流、一份電力合約、一紙政府命令）出事，連鎖反應會比大家想的更猛。便宜跟自主很少能同時拿滿；當你為了規模去鎖死供給，你也就把自己的命運交到了那條供給鏈最脆弱的一節手上。這些數字現在被當成實力的證明在報導，但實力跟脆弱，常常是同一件事的兩面。

五、OpenAI 推 Daybreak，跟 Anthropic 在資安市場正面對撞

不全是錢的事，但也離不開錢。OpenAI 推出了一個叫 Daybreak 的資安計畫，把 GPT-5.5 系列模型跟 Codex 的安全能力綁在一起，自動化威脅建模、漏洞辨識、打補丁、修復這一整套流程。位置很明確——這是在對標 Anthropic 主打防禦型資安的 Project Glasswing，兩家在 AI 資安這塊正面開打。

兩個前沿實驗室同時把火力轉向資安，這個方向本身就值得記一筆。

它說明一件事：純拼模型誰更聰明的階段，邊際報酬正在遞減，於是大家開始往「把能力包裝成一個能收企業大錢的具體場景」轉，資安就是其中油水最厚的一塊。對使用者來說，這當然是好事，多了強力的防禦工具。但有個前提別忘了——當「找漏洞、打補丁」這件事被高度自動化，攻擊方手上拿的是同一套技術。AI 把找漏洞的成本壓到趨近於零，這把刀從來是雙面的：你這邊補得快，那邊挖得也快，而補丁要走完測試發版部署得用週算，逆推一個補丁做出攻擊只要幾天。資安自動化真正改變的，不是攻防誰贏，是兩邊都被按下了快轉鍵。

原文來源：OpenAI News

把帳單、定價、算力租約、資安投入這幾件事擺在一起，很容易得出「AI 變貴了、好日子結束了」的結論。但這個結論其實不太對。

便宜的東西變貴，本身不是風險——市場本來就會替免費的午餐找回成本，這很正常。真正該擔心的，是你在它還免費、還吃到飽的那段蜜月期裡，悄悄把整套工作流綁死在某個你完全沒有定價權的工具上。功能可以比較，速度可以 benchmark，但「定價權在誰手裡」這件事，平常沒人放進評估表，偏偏它才是哪天真出事時，決定你能不能全身而退的那一項。所以下次評估一個 AI 工具，除了問它現在多強多便宜，記得多問一句：它漲價的那天——那天一定會來——我換得掉嗎？
