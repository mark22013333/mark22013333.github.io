Open CoDesign — 開源的 AI 設計工具，Claude Design 的本地替代方案

發表於2026-05-12|更新於2026-05-12|開發工具
|總字數:1.7k|閱讀時間:5分鐘|瀏覽量:

大多數 AI 設計工具都在解決一個錯的問題。它們花了大量精力讓「生成」變得更快，卻把你的資料、你的 API 用量、你的設計檔案全部鎖進別人的雲端。你以為你在用工具，其實你在餵養平台。

Open CoDesign 走了相反的方向。

先講它是什麼

一句話：自己架的 Claude Design。打開桌面應用，用自然語言描述你要什麼畫面，幾秒鐘後拿到一個可互動的 HTML prototype。聽起來跟 Claude Artifacts 或 v0 很像？差別在於——模型你自己選、API key 你自己帶、所有資料存在你的硬碟裡、沒有任何 telemetry 回傳。

這個專案 2026 年 4 月才公開，18 天衝上近 5,000 顆星。主要開發者是中國的 OpenCoworkAI 團隊，技術棧是 Electron + React 19 + Vite 7 + TypeScript strict mode，用 Zustand 管狀態、Biome 做 linting、Vitest 跑測試、Playwright 跑 E2E。

如果你之前用過 Claude Design 或 Bolt.new，體驗上非常接近。但帳是算在你自己的 API key 上。

BYOK 這件事為什麼重要

想像你在餐廳吃飯。多數 SaaS 設計工具的模式是「吃到飽月費制」——不管你這個月做了一張圖還是一百張，價格一樣。聽起來划算，但如果你只是偶爾需要快速出個 prototype，等於每個月花錢養一把很少打開的瑞士刀。

Open CoDesign 是「自帶食材到廚房」。BYOK（Bring Your Own Key）意味著你接上 Claude、GPT、Gemini、DeepSeek，甚至本地跑 Ollama，用多少算多少。支援超過 20 種模型，換模型就改個下拉選單的事。

這不只是省錢的問題，而是控制權。你的 prompt、你的設計產出、你的品牌素材，全部留在本機。對於接案的設計師或處理客戶敏感資料的團隊，這個差異是根本性的。

內建的設計技能不是噱頭

Open CoDesign 底層跑的是基於 pi-coding-agent 的 agent 迴圈，但讓它跟一般 AI coding assistant 拉開距離的是 12 個 Design Skill Modules。你可以把這些 skill 理解成「預訓練好的設計專家」：有專門做 Slide Deck 的、專門蓋 Dashboard 的、專門切 Landing Page 的。

選好 skill 之後，agent 就會帶著對應的 system prompt 和設計約束去生成。產出品質比你丟一段裸 prompt 給 ChatGPT 好一截，因為 skill 裡已經編碼了排版比例、色彩對比、元件間距這些設計基本功。

15 個內建 Demo prompt 也值得玩一輪，像是「SaaS pricing page」或「analytics dashboard」，直接看它能做到什麼程度。

Comment 模式和 AI Sliders

這兩個功能是我覺得做得最聰明的設計。

Comment 模式：prototype 產出後，你不用回去改 prompt。直接在畫面上點某個按鈕、某段文字，留下修改指示——「這個按鈕改成綠色」「這段標題字放大」。Agent 會針對你標記的元素去改，不是整份重生。這個互動方式比「重新描述整個畫面」高效太多了。

AI Sliders 更直覺。拉桿調整顏色飽和度、元件間距、字型大小，即時反映在 prototype 上。不用懂 CSS，不用寫 padding: 24px，拉一下就完事。

對，它也支援 DESIGN.md——你可以定義品牌色、字型、間距規範，之後每次生成都會遵守這套系統。這對需要維持視覺一致性的專案來說，省下的來回修改時間非常可觀。

匯出格式很實際

產出的 prototype 可以匯出成 HTML、PDF、PPTX、ZIP。

HTML 直接丟給工程師看互動效果。PDF 給老闆看靜態版面。PPTX 拿去提案。ZIP 打包整份原始碼。四種格式涵蓋了從「我要 demo 給客戶看」到「我要交給前端工程師做」的完整場景。

有一說一，能直接輸出 PPTX 這點蠻實用的。很多 prototype 工具到了要提案的時候，你還是得手動截圖貼 PowerPoint，Open CoDesign 直接幫你跳過這步。

安裝就一行

macOS 用 Homebrew：

Windows 用 winget 或 scoop 都行。需要 Node.js 22 以上。如果你想從原始碼建置也可以，畢竟整個專案是 MIT 開源的。

裝好之後填入你的 API key，選個模型，就能開始用了。沒有註冊流程、沒有信用卡綁定、沒有 onboarding 問卷。

但它不是萬能的

幾個你該知道的限制。

目前是 v0.1.x，安裝檔還沒有程式碼簽名，macOS 會跳出「無法驗證開發者」的警告，你得手動到系統設定裡允許。這對非工程師的使用者來說是個門檻。

它不是 Figma 的替代品。沒有向量編輯、沒有圖層概念、沒有多人即時協作。它做的事情比較像是「快速把想法變成可互動的 HTML 骨架」，而不是「精雕細琢每個像素」。

沒有 undo。對，2026 年了還沒有 undo。改壞了就重新生成，或者從 comment 模式局部修正。這一點需要習慣。

最後，產出品質高度依賴你選的模型。Claude 3.5 Sonnet 的前端生成能力明顯比其他模型好，如果你用比較小的模型，結果可能會讓你失望。

這件事背後的兩條線

AI 設計工具這個領域，正在沿著兩條線分裂。

一條是平台線：Claude Design、v0、Bolt.new——體驗打磨得很好，但你的一切都在別人的伺服器上。你付月費，你用他們的模型，你的設計留在他們的雲端。方便，但你沒有真正擁有任何東西。

另一條是工具線：Open CoDesign 走的這條——你帶自己的 key，跑自己選的模型，檔案存自己的硬碟。粗糙一點，要自己組裝一點，但每一個 bit 都是你的。

這不是哪條線比較好的問題。而是你得搞清楚自己需要的是「服務」還是「工具」。如果你只是偶爾出個 prototype 看看效果，平台線完全夠用。但如果設計產出是你的核心工作流、如果你處理的資料不能離開本機、如果你不想被單一供應商的定價綁住——那工具線的價值就很明確了。

Open CoDesign 還很年輕，功能上有不少缺口。但它證明了一件事：AI 設計工具不一定要是 SaaS，不一定要月費，不一定要把你的資料送上雲端。光是這個存在本身，就已經在改變遊戲規則了。
