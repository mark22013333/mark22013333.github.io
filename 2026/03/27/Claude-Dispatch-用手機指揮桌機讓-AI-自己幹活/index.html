Claude Dispatch 完整指南 — 用手機指揮桌機，讓 AI 在你電腦上自己幹活

發表於2026-03-27|更新於2026-04-13|Claude Code
|總字數:1.3k|閱讀時間:4分鐘|瀏覽量:

3 月 17 日，Anthropic 發佈了 Dispatch。一週後，3 月 24 日，Computer Use 跟著上線。

這兩個功能合在一起，讓 Claude 可以接受你手機丟過來的任務，然後自己在你的桌機上開 App、點按鈕、填表單、查瀏覽器、存檔案——你不用守在電腦前面。

Dispatch 是什麼

最直白的類比：對講機。你的手機是按鍵發話端，桌機是執行端。

你在外面（或者躺在床上），用 Claude 手機 app 發訊息：「幫我把今天的 PR review 整理成摘要，寄給我」。Dispatch 把這個任務送到你的 Mac，Claude Cowork 開始幹活——開 GitHub、讀 PR、整理、寄信。跑完通知你。

不是把任務雲端化，是真的在你的電腦上執行，用的是你電腦上已經設定好的連接器、工具、帳號。

Computer Use 是什麼

Claude 優先使用有直接整合的工具（Slack connector、Google Calendar 之類的 MCP 工具）。但如果工具不夠用呢？

這時候 Computer Use 上場。Claude 用截圖看你的螢幕，然後控制滑鼠和鍵盤——點按鈕、輸入文字、切換視窗。就像一個遠端操控你電腦的 AI 助手，但它看到的是螢幕畫面，不是直接存取 API。

官方說法是「工作方式跟人類使用電腦一樣」。實際上就是：Claude 截圖 → 判斷要點哪裡 → 發出點擊指令 → 再截圖確認結果 → 繼續。

怎麼開啟

前提條件：

  • Claude Pro 或 Claude Max 方案
  • macOS（目前僅限）
  • Claude Desktop App 最新版本

開啟步驟：

  1. 打開 Claude Desktop App
  2. 進入設定（⌘,）
  3. 找到 Cowork 相關設定，開啟 Computer Use
  4. 確保 Desktop App 在執行中（不能是最小化或休眠）

手機端：

下載 Claude 手機 app，登入同一個帳號。Dispatch 功能在對話介面裡，可以指定在桌機執行任務。

實際使用情境

情境一：外出時交辦繁瑣任務

適合那種「回到家再說」但其實會忘記的事情。

情境二：自動化有 GUI 的工作流

有些工具沒有 API，只有 Web 介面。Computer Use 可以操作這類工具。

比如說某些公司內部系統只能在瀏覽器裡操作：「幫我在 [內部系統] 提交 3 月份差旅報銷，金額是 XXX」，Claude 自己去操作那個只有人工界面的系統。

（前提是你要先開著那個系統的視窗，Claude 才看得到）

情境三：讓 Cowork 排程任務在背景執行

Cowork 支援 /loop 排程指令。搭配 Dispatch，你可以設定讓 Claude 定期（比如每天早上 8 點）執行某個任務，你不需要在場。

運作機制與安全邊界

幾個值得知道的細節：

工具優先級：Claude 先用現有連接器（有 MCP 整合的工具），連不到才切換 Computer Use。這個設計讓操作更穩定，畢竟直接 API 比視覺操控可靠。

Prompt Injection 防護：Anthropic 有掃描每次操作，偵測畫面上是否有嘗試「改變 Claude 行為」的文字（prompt injection 攻擊）。

需要明確授權：Claude 在存取新應用之前需要你的明確允許。部分應用預設是禁止存取的（基於安全考量）。

桌機要開著：Desktop App 必須在執行中，才能接收 Dispatch 任務。手機 app 會提示你桌機是否在線。

目前的限制

官方說法很坦誠——「這還是早期階段」：

  • 比有直接整合的工具慢（視覺截圖 → 操控 → 截圖確認，每個步驟都有延遲）
  • 可能犯錯，尤其是 UI 比較複雜或變化的時候
  • 目前只支援 macOS
  • 部分應用基於安全原因禁止存取

Computer Use 這類功能的本質就是「讓 AI 看螢幕然後動手」，現在的準確度已經能做很多事，但不要把它當成萬無一失的自動化方案。

跟 Claude Code Remote Control 的差異

上個月介紹過 Claude Code Remote Control（手機遙控終端機），這次的 Dispatch 跟它的差別在於：

Claude Code Remote Control Dispatch + Computer Use
執行環境 終端機 / Claude Code Claude Cowork（桌面 App）
操控對象 命令列工具 任何 App、瀏覽器、GUI
主要用途 開發任務、程式碼 一般電腦操作任務
目標族群 開發者 所有 Pro/Max 用戶

簡單說：Remote Control 是工程師版本，Dispatch 是給所有人的版本。

小結

Dispatch + Computer Use 讓「AI 幫我用電腦」從概念變成實際可用的功能。不用設 API、不用串接，直接描述任務就好——它自己去點。

上頭的感覺是有，特別是第一次看著 Claude 自己在你螢幕上開瀏覽器、切視窗、填表單的時候。但也要記住它現在的限制，給它做比較簡單、重複性高、後果不嚴重的任務，比較穩。
