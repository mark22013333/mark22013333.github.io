AI 與科技新聞摘要 - 2026/04/23

發表於2026-04-23|更新於2026-04-23|AI與科技新聞

$1,000,000,000,000。

這不是 Anthropic 的融資額，是它在二級市場的估值。4 月 23 日，Anthropic 在 Forge Global 等二級市場交易平台的隱含估值越過了兆美元門檻，同時間 OpenAI 停在 $8,800 億，低於它自己去年底 $8,520 億的官方估值。

這個數字的荒謬感在於，Anthropic 連 IPO 都還沒有。公開市場對它完全無從定價。但機構投資人搶得如此激烈，把二級流通股票的要價推到了傳統估值方法難以支撐的位置。這不是理性定價，是稀缺性恐慌——害怕現在不買，以後根本買不到。

同一週，Google Cloud Next 2026 在拉斯維加斯展示了一套重新架構的 agent 基礎設施，A2A 協定正式量產，支援者從六個框架變成幾乎整個生態系。

有些事情正在同時改變，它們之間有關聯。

Google Cloud Next 2026 — Vertex AI 不見了，Gemini Enterprise 接手

4 月 22 日 Google Cloud Next 2026 最大的一個動作，不是發布新功能，是改名。

Vertex AI 正式更名為 Gemini Enterprise Agent Platform，Agentspace 也被吸進同一個統一產品下。在外界看來像是整合，在 Google 內部的戰略意圖是更清楚的訊號：Google 在企業 AI 市場想要的不是「平台上的一堆工具」，是「一個端到端的 agent 基礎設施」。

這次發布的具體內容包括 Workspace Studio（無程式碼 agent 建構器）、Model Garden 現在掛了超過 200 個模型（含 Anthropic Claude、Llama、Gemma）、以及來自 Box、Workday、Salesforce、ServiceNow 的合作夥伴 agent。

但真正值得看的是 A2A v1.0 正式進入量產。

A2A（Agent-to-Agent Protocol）從去年的實驗規格，在這次確認已經有 150 個組織在生產環境中跑。更重要的是，它的支援者清單變成了：Google ADK v1.0、LangGraph、CrewAI、LlamaIndex Agents、Semantic Kernel、AutoGen——幾乎涵蓋所有主流 agent 框架。

用具體例子說：一個 Salesforce agent 可以把任務遞交給跑在 Google Vertex 的 agent，後者再查詢 ServiceNow 的 IT 資產資料，整個流程三個系統不需要理解彼此的內部架構，只靠 A2A 溝通。

這是 MCP 解決「AI 跟工具怎麼連」的問題，A2A 解決「AI agent 跟另一個 AI agent 怎麼協作」的問題。兩個協定現在都在快速落地，而且方向是配套的而不是競爭的。

逆向想這件事：Google 為什麼願意讓 A2A 成為開放標準、讓 CrewAI 和 LangGraph 也原生支援？因為 Google 的 agent 生態系愈大、標準愈廣，跑在 Google Cloud 上的 agent workload 就愈多。它不是在做公益，是在用開放標準換基礎設施黏性。

Anthropic 二級市場突破 $1 兆估值，超越 OpenAI

4 月 23 日，複數個二級市場平台的 Anthropic 交易數據越過了 $1 兆美元門檻，比 OpenAI 的 $8,800 億多出大約 14%。

背後的財務數字有幾個：

  • 年化營收 ARR：Anthropic $300 億 vs OpenAI $250 億（2026 年 3 月對比）
  • 季增速：Anthropic 從 2025 年底 $90 億跳到 $300 億，單季增長 233%
  • 企業客戶每年付超過 $100 萬的：500 家（2 月）→ 1,000+ 家（4 月初），兩個月翻倍
  • Claude Code 年化營收：超過 $10 億，上線六個月達到

OpenAI 的問題不是它停止成長，是 Anthropic 成長更快。

有幾個結構性因素在背後。第一，Claude 在企業 API 和開發者工具這個細分市場做得比 ChatGPT 更好——不是因為聊天能力更強，是因為 API 設計、tool use 穩定性、長 context 管理這些讓工程師用起來更順手的地方做得更紮實。第二，Claude Code 在 AI coding 爆炸性成長的這個波段搭上了。第三，Anthropic 的企業銷售能力在過去一年大幅提升，從「AI 安全公司」的品牌定位，順利轉型成「企業 AI 基礎設施」。

兆美元估值的下行風險在哪？二級市場的稀缺性溢價不等於公開市場的合理估值。Anthropic 自己的 IPO 目標估值是 $4,000 億到 $5,000 億——比現在的二級市場價格低了一半。如果上市定價在這個範圍，現在在二級市場接盤的投資人會拿到一個帳面損失。這種結構在科技泡沫史上不是第一次出現。

Claude Code v2.1.118 — vim 視覺模式 + 自訂主題

4 月 23 日釋出的 v2.1.118 有兩個面向使用者的更新。

第一個：vim visual mode 和 visual-line mode 正式上線。在 Claude Code 的終端機輸入介面裡，現在可以用 v 進入字元選取模式、V 進入行選取模式，搭配 operators 做複製、刪除、替換。對習慣 vim 操作流程的開發者來說，這個改變讓在終端機裡編輯長段 prompt 的體驗好了不止一個檔次。

第二個：named custom themes。可以用 /theme 指令建立和切換自訂主題，主題設定有名稱，可以存下來在不同 session 之間切換。顏色方案是終端機工具非常個人的設定，現在不需要每次 session 重設了。

這兩個更新單獨看都很小。但過去五週，Claude Code 發布了超過 30 個版本：v2.1.90 的 NO_FLICKER 渲染引擎讓 prompt 渲染循環少了 74%、Write tool 的 diff 計算快了 60%、記憶體佔用降了 80MB。每個版本都是小修，積累起來是一個工具在細節上的成熟。

原文來源：Claude Code Changelog - code.claude.com

Anthropic 與 Google、Broadcom 簽 3.5GW TPU 供應協議

和估值新聞一起釋出的，還有一個計算資源協議的更新。

Anthropic 宣布與 Google 和 Broadcom 簽訂新協議，Google 將從 2027 年起提供約 3.5GW 的下一代 TPU 算力，加上已承諾的 2026 年約 1GW。合計約 4.5GW 的 TPU 算力。

這個數字值得換個單位來感受一下：3.5GW 大約是一個中型核電廠的輸出功率，或者約 250 萬個家庭的用電量。這不是一個正常的雲端算力採購，是把一整個計算基礎設施綁在一起。

從 Anthropic 的角度，這解決了模型訓練最大的不確定性：算力能不能拿到。從 Google 和 Broadcom 的角度，這是把 Anthropic 的未來成長牢牢綁進自己的硬體生態。

原文來源：Anthropic Just Announced Huge News for Alphabet and Broadcom - The Motley Fool

把今天幾條新聞串起來

表面上看是估值、改名、版本更新，拉長時間軸看是同一件事在不同層次上展開。

AI agent 基礎設施的三個層：model（Anthropic、OpenAI、Google 的競爭）、protocol（A2A + MCP 的標準化）、tooling（Claude Code、Cursor、Copilot 的功能競爭）——這三層現在同時在快速演化，而且彼此緊密耦合。

A2A 量產、Agent Skills 提供工程紀律、Claude Code 打磨細節。這些不是獨立事件，是同一個方向的不同切面：AI agent 從「可以演示」變成「可以在生產環境跑」的轉型過程。

預測誰贏誰輸是沒意義的。有意義的問題是：在這個轉型過程裡，哪些決策是可逆的，哪些是不可逆的。採用哪個 protocol 作為企業基礎設施，通常是不可逆的。這個判斷現在比任何估值數字都重要。
