AI 與科技新聞摘要 - 2026/04/27

發表於2026-04-27|更新於2026-04-27|AI與科技新聞

1,000 份解僱通知。同一份公告裡寫著：AI 現在寫了 65% 的新 code。

Snap 在 4 月 15 日裁掉 16% 的員工。CEO Evan Spiegel 說這是「AI 帶來的新工作方式」，預計下半年省下超過 $5 億美元營運成本。股價？當天上漲。

這不是效率故事。這是一面鏡子——照出來的是整個軟體產業正在經歷的結構性位移。同一週，JetBrains 的萬人調查說 74% 開發者已經在用 AI coding 工具，Kiro CLI 推出 2.0 正式支援 headless 模式，而 Stanford 的年度報告寫著：AI agent 在真實電腦任務的成功率從 12% 跳到 66%。

每一條單獨看都是一則新聞。放在一起看，是一整個系統在加速重組。

Snap 裁員 1,000 人：AI 寫 65% 的 code，人寫剩下的

4 月 15 日早上，大約 1,000 名 Snap 員工收到了解僱信。

Spiegel 的內部信很直白：AI 讓更小的團隊能做更多事。Snap 把更多「非核心」的 coding 工作丟給 AI agent，保留的人力集中在高判斷力的任務——架構決策、使用者體驗、系統設計。

65% 這個數字本身不是重點。重點是一家 $200 億市值的上市公司，對外公開說「AI 取代了我們一大塊工程產能」，而市場的反應是正面的。這跟兩年前的敘事完全不同——2024 年如果有公司說「AI 寫我們的 code」，分析師的第一反應是質疑品質。2026 年，分析師的反應是「你們 AI 化得還不夠快嗎？」

逆向問：如果 AI 寫了 65% 的 code 就能裁 16% 的人，那 AI 寫到 85% 的時候呢？數學不是線性的——65% 到 85% 裡面包含的是越來越高判斷力的任務。但壓力方向很清楚。

原文來源：Snap to Cut 1,000 Jobs, Have AI Take On 65% of Coding
原文來源：Snap is cutting 1,000 jobs, 16% of its workforce | TechCrunch

JetBrains 調查：74% 開發者已在用 AI coding 工具

JetBrains 在 2026 年 1 月跑了第二波 AI Pulse 調查，8 種語言、超過 10,000 名職業開發者。

核心數字：74% 的開發者已經在工作中使用 AI coding 工具。不是 ChatGPT 這種通用聊天機器人，是專門設計給開發者的——AI coding assistant、AI-native 編輯器、coding agent。

工具知名度排行：GitHub Copilot 76% 的開發者聽過，29% 在用。Claude Code 57% 聽過，18% 在用（比六個月前增加了 50%）。Google Antigravity 在 2025 年 11 月上線後，兩個月內衝到 6% 使用率。

這組數字讓人在意的不是誰領先——Copilot 先發優勢明擺著在那裡。在意的是增速。Claude Code 半年內使用率增加 50%，代表的是什麼？是 terminal-native 的開發者偏好正在從邊緣走向主流。六個月前，「在 terminal 裡用 AI 寫 code」還是少數人的玩法；現在快兩成開發者把它當日常工具。

如果 Snap 的裁員是供給面的訊號（公司端在導入 AI），JetBrains 的調查就是需求面的訊號（開發者端也在主動擁抱 AI）。供需同時加速，轉折點過了。

原文來源：Which AI Coding Tools Do Developers Actually Use at Work? | JetBrains

Kiro CLI 2.0：headless 模式、Windows 支援、任務面板

AWS 的 Kiro 團隊在 4 月 13 日推出 CLI 2.0，三個主要功能：headless 模式、Windows 原生支援、重新設計的 Terminal UI。

headless 模式讓 Kiro 可以跑在 CI/CD pipeline 裡——用 KIRO_API_KEY 認證，加 --no-interactive 參數，Kiro 就會端到端執行你的 prompt，不需要人盯著。配合 --trust-all-tools 或 --trust-tools 可以預先授權工具權限。

這是什麼意思？AI coding agent 正在從「interactive 工具」走向「infrastructure 元件」。當你可以在 GitHub Actions 裡跑一個 Kiro headless 任務，它就不再是「輔助你寫 code 的工具」，它是「pipeline 裡的一個自動化節點」。跟 Claude Code 的 --headless 和 --allowedTools 是同一個方向。

Windows 支援是另一個值得注意的訊號。之前 Kiro CLI 只跑在 macOS 和 Linux 上，Windows 開發者——全球最大的開發者群體——被排除在外。現在 Windows 11 完整支援 Terminal UI、headless mode、MCP server、custom agent，等於 Kiro 的潛在用戶池瞬間翻倍。

怎樣做一定失敗？只支援一個平台、只支援 interactive 模式。Kiro 2.0 同時補了這兩個短板。

原文來源：Kiro CLI 2.0: a new look and feel, headless CI/CD pipelines, and Windows support

Shield AI $1.5B Series G：國防 AI 估值 $12.7B

Shield AI 拿了 $1.5B 的 Series G，整個資本包（含債務）是 $2.25B，估值 $12.7B——比一年前翻了 1.4 倍。

Shield AI 做的是 Hivemind 自主飛行平台，已經被美國空軍選為 Collaborative Combat Aircraft（CCA）計劃的技術提供商。白話翻譯：讓無人機自己飛、自己判斷、自己做戰術決策。

為什麼一個國防 AI 公司的估值在一年內翻倍？因為全球軍事預算正在系統性地往 AI 移動。不只是美國——歐洲、亞太、中東都在追。AI 國防的投資節奏跟 AI 商用軟體不同：一旦簽了合約，通常是 5-10 年的承諾，不會因為一季財報不好就砍預算。

風險在哪裡？自主武器系統的監管壓力才剛開始。歐盟 AI Act 的高風險系統分類裡，軍事應用是最敏感的區塊之一。Shield AI 的估值假設是「市場持續擴張」，但如果國際監管收緊，擴張的速度可能比預期慢。

$12.7B 估值的底層假設是：各國政府會持續加碼 AI 軍事預算，而且 Shield AI 能守住技術領先。這兩個假設在今天都成立。問題是五年後呢？

原文來源：AI News Digest, April 26, 2026: Defense AI Hits $12.7B as Funding Gravity Shifts

Stanford AI Index：AI Agent 成功率從 12% 跳到 66%

Stanford 的 2026 AI Index 報告出了。其中一個數據特別值得拿出來看：AI agent 在真實電腦任務上的成功率，一年之內從 12% 跳到 66%。

「真實電腦任務」是什麼意思？是讓 AI agent 操作軟體、瀏覽網頁、管理檔案——不是 benchmark 上的受控測試，是實際的日常電腦操作。一年前只有 12% 能完成，現在接近七成。

這個增速如果持續，12 個月後的數字會是什麼？不要做精確預測——精確預測本身就是一種傲慢。但方向很清楚：AI agent 離「完成人類日常電腦操作」這件事，已經從「不太行」走到「大部分時候可以」。

把這個數字跟 Snap 的 65% AI coding、JetBrains 的 74% 採用率放在一起看：供給端（AI 能力）和需求端（人類採用意願）同時在加速。去年的 AI agent 還在「展示可能性」的階段，今年已經進入「處理實際工作流」的階段。

這不是漸進式改良。從 12% 到 66% 是相變——物質的狀態改變了，不是溫度升高了一點。

原文來源：2026’s Q1 AI Updates - AI Agents Simplified

GitHub 啟動 Token 格式遷移：4/27 今天開始

從今天（4 月 27 日）開始，GitHub 開始分階段更新 GitHub App installation token 的格式。新格式更長、無狀態、效能更好。

影響範圍：GitHub Actions 的 GITHUB_TOKEN、Dependabot、Slack、Teams 等一方整合的 token 都在這波。完整的第三方 App installation token 遷移預計 5 月中到 6 月底完成。

如果你的 CI/CD pipeline 或自動化腳本有硬編碼 token 長度檢查（例如「token 必須剛好 40 個字元」），今天開始可能會壞掉。GitHub 說大部分工作流不受影響，但如果你遇到問題可以暫時 opt-out。

這是一個「小概率但高影響」的變更。99% 的人不會受影響，但那 1% 如果剛好是你，你的整條 CI/CD 管線可能突然斷掉，而且錯誤訊息可能不會直接告訴你是 token 格式的問題。

原文來源：Notice about upcoming new format for GitHub App installation tokens - GitHub Changelog

這六條新聞的底層結構有一個共同的特徵：不可逆性。

Snap 不會把裁掉的 1,000 人再請回來。74% 的開發者不會回頭用純手寫 code。AI agent 的成功率不會從 66% 掉回 12%。GitHub 的 token 格式不會改回去。

可逆的變化是波動。不可逆的變化是趨勢。波動讓你興奮或焦慮，趨勢讓你調整位置。

現在要問的問題不是「AI 會不會改變軟體工程」——那個問題已經被回答了。現在的問題是「你在這個結構性位移裡站在哪裡」。
