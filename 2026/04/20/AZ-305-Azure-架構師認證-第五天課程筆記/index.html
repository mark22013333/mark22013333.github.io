AZ-305 Azure 架構師認證 - 第五天課程筆記

發表於2026-04-20|更新於2026-05-14|微軟課程
|總字數:3.8k|閱讀時間:11分鐘|瀏覽量:

**20%**。這是英語版 AZ-305 題庫在 2026-04-17 更新的變動比例。新題範疇集中在監控、備份、遷移——也就是第四天才剛上完的東西。繁中、簡中、港版會在兩到三週內陸續同步，6 月底前完成。

想像你準備某科期末考準備到一半，突然老師改了五分之一的題目——這大概是大部分考生最怕聽到的消息。第五天講師開場就先打這劑強心針：怎麼把這個變動納進備考時程。

這篇是 AZ-305 第五天的課堂筆記。前四天講技術，這一天全部講「怎麼考過」——從報名、考場選擇、費用優惠、到六題範例解析，整個考試流程被壓在最後一天過完。

考試定位：兩張考才拿得到專家級

AZ-305 是 Azure Solutions Architect Expert 三星專家級認證。這個級別的設計是：必須同時通過 AZ-305 跟 AZ-104（Azure Administrator Associate），微軟才會核發專家證書。

順序不限。可以先考 AZ-305 再補 AZ-104，也可以反過來。但只有一張的話，只會拿到對應的單一認證，不會有專家級徽章。

這個設計的邏輯很清楚：架構師不能只會畫架構圖，還得懂基礎操作。AZ-104 證明你會開機、會設網路、會管帳號；AZ-305 證明你能把這些零件組成解決方案。兩張加起來才是完整的架構師。

認證入口：從電子書找進去

入口在 Microsoft Learn 的電子書：「設計 Microsoft Azure 基礎結構解決方案」→ 右上角「相關認證」。

進去之後有個小技巧——網址最後的 zh-cn 改成 zh-tw 就是繁中、改 zh-hk 就是港版。內容完全一致，只是翻譯的用詞風格不同。中港台三個版本各有偏好，挑看得最順的即可。

關於那個英語版題庫變動 20% 的消息，有個訊號值得注意：其他語系的同步要等兩到三週。這段期間如果你考的是英語版，建議把第四天（監控、備份、遷移）的內容再加強一輪，因為新題範疇就在那邊。考繁中、簡中、港版的話，最好等同步完再考；除非你 5 月上旬就必須考完，否則緩個兩週壓力會小很多。

註冊帳號：這條地雷很大

強烈建議用個人帳戶註冊（Hotmail / Outlook 等）。絕對不要用公司帳戶。

原因是這樣的——微軟認證綁的是帳戶，不是人。如果你離職，公司帳戶會被 HR 或 IT 收回，你花了 83 美元考到的證書就一起被收回去了。微軟無法回復這類認證，最多就是個客服工單跑流程最後被駁回。

基本資料也有坑。地址、城市務必填真實且為英文：微軟極低機率會寄送實體禮品卡，中文地址在海外物流系統轉不出來。

右下角的 Email 欄位是收取報考確認信的唯一管道、也是報考成功的唯一憑證——填錯一次就沒補救方法，必須重開帳號。

費曼式的檢查清單：認證就像你的結婚證書，上面的名字要跟身份證一致，收件地址要真的能收到信，帳號要是你離職後還保得住的那個。這三條沒做對，後面都是白忙一場。

考場選擇：線上還是實體？

微軟認證統一由 Pearson VUE 代理。兩種應考方式，各有坑。

線上考試 OnVUE：自家電腦安裝軟體、自備環境。關鍵細節：

  • 監考員同時監考多場，不是專職盯你
  • 網路斷線、鏡頭看不到人 → 直接判定作弊、中止考試，費用不退
  • 伺服器在海外，中國大陸考生網路風險高
  • 適合：有穩定海外連線、考過多次的熟練考生

實體考試中心（講師強烈建議首考者用這個）：

  • 台灣（台北、桃園、台南、花蓮等）、香港（多區）、中國大陸省會級城市基本都有
  • 偏遠地區會例外（例如貴州貴安新區大數據中心區）
  • 一次最多可勾選 3 個考場，不同考場的開放日不一

選考場有四個訣竅：

  1. 區域優先：離自己最近的考場當主選，但同時勾選 2-3 個備案
  2. 時間靈活：主考場時段不合適就換稍遠但週末或工作日可選的
  3. 避開熱門時段：單一考場通常 4 台機器，上午場人滿；別人先考完離場會給你心理壓力
  4. 建議下午 10:00～11:00 場：人少、安靜、不被干擾

考場列表裡黃底可展開代表該日尚有其他時段，點「瀏覽更多時間」就能看到。

改期、取消、費用：規則比想像寬鬆

免費改期、取消各 1 次機會，共兩次操作空間。必須在考試開始前 24 小時以上完成。24 小時內無法改、遲到無法補救。

千萬不要遲到。遲到等於考試報廢，無法改簽、也無法用免費券彌補。

費用這邊有個要點：

  • 原價 83 美元（USD 計費，不是人民幣）
  • ESI 優惠：公司加入微軟 ESI 計畫，個人帳號綁定後自動帶入 50% 折扣
  • 免費考試券：取消考試後會退回，可用於下次預約

關鍵操作細節：使用免費考試券前必須刪除 ESI 折扣。考試券沒辦法對打完折的殘額再折，只能扣原價。理想目標是報名總價歸零（Total USD 0.00）。

證件要求：帶有照片的身份證件（身份證 / 護照 / 工作證都可，但必須有照片）。註冊時只留姓名、沒有臉部辨識，到考場時靠照片證件核對本人。無照片一律不讓進。

建議考試時程：今天（2026-04-17）起 1 個月內（5/16 前）考完最佳。一個月是剛好的複習窗口——太早寫不完，太晚會忘。

證書效期：1 年 + 免費 Renew

專家級認證有效期是 1 年。有效期過半（約 6 個月）時微軟會寄信提醒續訂。

續訂不用重新註冊或付費：

  • 登入認證頁面 → 點「Renew」→ 線上作答約 26 題
  • 通過後自動延展 1 年

這個設計對持續學習者很友善。你通過一次之後，只要每年花一兩小時回頭複習、做 26 題線上測驗，證書就能一直續下去。

複習策略：官方資源才是金礦

核心原則：尊重官方、以官方為主。不要迷信外部傳言或付費題庫，官方資料才是最珍貴的備考來源。

官方資源就兩個：

  1. 電子書（Study Guide）：從頭到尾至少讀 1～2 遍，每章都要讀完
  2. 免費練習評估（Practice Assessment）：50 題，不會出現於考試，但能驗證知識點

就這兩個。沒有官方備考影片——AZ-305 知識點太廣、橫跨多個服務，微軟沒做專門影片。所有「一週速通 AZ-305」之類的 YouTube 內容都是第三方，參考就好。

官方公佈的知識點佔比跟課程安排略有差異，實際體驗大致是：

  • 基礎結構（計算、網路、儲存、資料庫）→ 最大頭，約 60%
  • Identity（Microsoft Entra ID）+ 治理（Policy / RBAC）→ 約 15~20%
  • 業務連續性（備份、遷移、監控）→ 佔比最少，但每次都會出

側重順序建議：

  1. 計算、網路、儲存、資料庫（Day 1～Day 3 的內容）
  2. Identity、RBAC、Azure Policy
  3. 備份與遷移（Day 4 內容）——佔比少但必考

還有個容易忽略的點：AZ-305 屬於 AZ-104 的進階延伸，題庫含有基礎操作層級的題目（如基本使用者管理）。複習時可以回頭看 AZ-104 內容做補強，別因為你是「架構師」就跳過操作細節。

題型與評分：微軟相對寬容

題數：每人 50～60 題（隨機抽題，每人題目不一樣）。滿分 1000，700 分合格。時間通常 1 小時內可完成。

所有答案都在螢幕上用滑鼠點選或拖拉，不需鍵盤打字。

題型共六種：

  1. 單選題：選項前為圓框
  2. 多選題：選項前為方框；題幹會明示「Select X」要選幾個
  3. 判斷題：Yes / No
  4. 下拉選單題（Dropdown）：題幹中插下拉框
  5. 熱點題（Hotspot / 拖拉）：左邊選項、右邊空格，有文字版與白框版
  6. 拖拉排序題（最難）：先挑出正確選項，再按正確順序排列

評分規則微軟算是寬容：

  • 單選：對就得分、錯不扣分
  • 多選：對一個給部分分數、漏選不滿分、多選錯的不倒扣
  • 比某些「選錯倒扣分」的測驗人性許多

題幹語言的坑：題幹、選項依選定的語系翻譯，但服務名稱（Azure 服務）、圖片內文字一律英文不翻譯。讀題時要留意圖片內的英文專有名詞，尤其是縮寫（VMSS、PPG、AOAG）和相似名稱（Availability Set vs Availability Zone vs Proximity Placement Group）。

做題技巧：倒著讀題法

長題幹加表格資料是 AZ-305 的特色。講師的倒著讀題法很值得學：

  1. 先讀最後的問題（真正要回答的那一句）
  2. 再讀答案選項（先框定待選空間）
  3. 最後回頭讀題幹（帶著問題比對條件）

這個方法的原理很費曼：題幹是被問題框住的資訊，先知道問題才知道哪些資訊重要、哪些是干擾項。反過來從頭讀，大腦會試著記住所有細節——考試時間根本不夠這樣讀。

兩個額外提醒：

注意單選 / 多選圖示。圓框只能挑一個，方框必須選滿正確數量才給分。搞錯會痛失整題的分。

不過度鑽牛角尖。官方出題偶有瑕疵（例如「刪除資源組」情境忽略 7 天冷卻期）。以官方視為「正確」的答案為主，不要在細節上鑽——實務上需要額外步驟，不代表考試就該選那個答案。

六題範例：建立考點敏感度

課程最後挑了幾題有代表性的範例。這些不是要背答案，是要建立看到關鍵字就觸發的反射。

範例 1：Proximity Placement Group

部署多台 VM，希望彼此之間網路延遲愈低愈好，應該選哪種功能？

✅ 正解：Proximity Placement Group

干擾項與為什麼不選：

  • Availability Set / Availability Zone：強調「高可用」或「跨區容災」，不是為了延遲
  • VMSS（Virtual Machine Scale Set）：解決「彈性擴展」，跟延遲無直接關係

反射規則：看到「低延遲 / low latency」就直接聯想到 Proximity Placement Group。

範例 2：資源組刪除受阻

嘗試刪除某 Resource Group，系統提示無法刪除。最常見的三種原因？

  • 資源組內含 Recovery Services Vault，其中仍存放備份資料
  • 資源組被設定了 Lock（CanNotDelete / ReadOnly）
  • 備份保險箱剛停用保護，尚在 7 天冷卻期內，無法立即刪除

反射規則：Recovery Services Vault 啟用「軟刪除（Soft Delete）」後，停用保護也會保留資料一段時間——急著刪資源組時常卡在這裡。

範例 3：Access Review 的適用對象

希望定期檢視外部合作單位的存取權限，選擇合適的 Entra ID 功能。

✅ 正解：Access Review（存取檢閱）

可針對 Guest 使用者、群組成員、應用程式指派做定期檢查。特別常用於管理 B2B Guest 帳號。

反射規則：關鍵字「定期檢視」+「外部使用者 / Guest」= Access Review。

範例 4：動態群組（Dynamic Group）

希望使用者依部門屬性自動加入對應群組，不想手動加人。

✅ 正解：Dynamic Group（規則式成員資格）

以屬性（如 Department、Country）組合條件自動納入成員。

干擾項：Assigned Group 需要逐一加入，不符合「自動化」訴求。

反射規則：要注意區分「Dynamic User」與「Dynamic Device」——題目會明確指定是使用者還是裝置。

範例 5：授予目錄角色（Directory Role）

需要授予使用者「管理其他使用者帳號」的權限，又不想給最高的全域管理員。

✅ 正解：User Administrator（使用者管理員）

可新增、修改、重設密碼等。

反射規則：AZ-305 會反覆測試「最小權限（Least Privilege）」。題目問誰最合適時，選那個剛好夠用、不多給的角色。

範例 6：SSO 存取地端 Web App

使用者在外面要以 Entra ID 帳號單一登入存取地端的 Web 應用，同時做條件式判斷。

✅ 正解：Application Proxy + Conditional Access

  • Application Proxy 把地端應用發佈到 Entra ID
  • Conditional Access 控管登入條件（例如必須 MFA、必須從企業網路）

干擾項與為什麼不選：VPN 雖可連線，但不屬於 SSO，也缺少 Conditional Access 的細部控制。

反射規則：關鍵字「地端 Web App + SSO + 外部存取」= Application Proxy。

補充參考網站

資源 角色
Microsoft Learn（電子書 + Practice Assessment） 最準、最貼近真題風格，官方出的一切都要看
Practice Assessment 官方模擬題，務必做
ExamTopics 大量題庫可練，但答案常與留言版討論相左；不要盲信單一來源，用社群投票+官方文件交叉比對
Microsoft Learn 官方文件 釐清服務差異時的最終裁判

ExamTopics 上若某題社群投票分歧，代表真題可能已更新或題意模糊，遇到這種題目以官方文件為準。

讀到這裡，可以開始報名了

前四天講技術，第五天講流程。技術會過時，流程不會。一年後你回頭看這份筆記，服務名稱可能已經改、介面可能已經換，但「倒著讀題」「選最小權限」「以官方為準」這些做事方式會一直用得到。

AZ-305 的真正價值不在證書本身——是它逼你把散落在各個專案裡的架構經驗，重新組織成一套有內在一致性的判斷框架。考試通過只是副產品。

學完這個可以接著學什麼？三個方向：

往深走：AZ-104（Azure Administrator Associate）補完專家級的另一半。這兩張都過了，才真的拿到「Solutions Architect Expert」徽章。

往廣走：AZ-400（DevOps Engineer Expert）或 AZ-204（Developer Associate）。前者處理 CI/CD 與自動化，後者處理實際寫 code 時的 Azure SDK 用法。

往 AI 走：AI-102（Azure AI Engineer Associate）或 DP-100（Data Scientist Associate）。AI 時代的架構師要懂 AI service 怎麼落地進企業架構。

背後就一條線——架構師的能力不是知道所有服務的名字，是在陌生情境下能快速挑出適合的組合。前四天教你有什麼零件，第五天教你怎麼證明自己會用。

願每位準備 AZ-305 的工程師，都能在一個月內拿到那 700 分。
