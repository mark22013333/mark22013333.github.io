еӨҡеҹ·иЎҢз·’з’°еўғдёӢжҺ§еҲ¶е…ұдә«иіҮжәҗзҡ„иЁӘе•ҸпјҢжҳҜжҜҸеҖӢ Java е·ҘзЁӢеё«еҝ…жңғзҡ„жҠҖиғҪгҖӮSemaphore жҺ§еҲ¶иЁұеҸҜж•ёйҮҸпјҢvolatile зўәдҝқеҸҜиҰӢжҖ§гҖӮйҖҷе…©еҖӢжҰӮеҝөеёёеёёиў«жҗһж··пјҢд»ҠеӨ©дёҖж¬ЎиӘӘжё…жҘҡгҖӮ

Semaphore еҹәжң¬жҰӮеҝө

Semaphore е°ұжҳҜгҖҢдҝЎиҷҹйҮҸгҖҚпјҢз”ЁдҫҶжҺ§еҲ¶еҗҢжҷӮиЁӘе•ҸжҹҗеҖӢиіҮжәҗзҡ„еҹ·иЎҢз·’ж•ёйҮҸгҖӮ

еҜҰжҷӮжҜ”е–»

еҒңи»Ҡе ҙжңү 10 еҖӢеҒңи»ҠдҪҚгҖӮжҜҸеҖӢйҖІдҫҶзҡ„и»ҠиҰҒе…ҲжӢҝеҖӢ ticketпјҲacquire иЁұеҸҜпјүпјҢеҒңеҘҪдәҶжүҚиғҪж”ҫ ticketпјҲrelease иЁұеҸҜпјүгҖӮеҸӘжңүжӢҝеҲ° ticket зҡ„и»ҠжүҚиғҪйҖІгҖӮ

еҹәжң¬з”Ёжі•

иҰҒй»һпјҡ

  • acquire() е’Ң release() иҰҒй…Қе°ҚпјҢйҖҡеёёж”ҫеңЁ try-finally
  • еҰӮжһңжІ’жңүиЁұеҸҜпјҢacquire() жңғйҳ»еЎһ
  • еҸҜд»ҘиЁӯе®ҡеҲқе§ӢиЁұеҸҜж•ёпјҲSemaphore(5) = 5 еҖӢиЁұеҸҜпјү

еӨҡеҹ·иЎҢз·’зҜ„дҫӢ

жіЁж„ҸпјҡеҗҢжҷӮжңҖеӨҡеҸӘжңү 2 еҖӢеҹ·иЎҢз·’еңЁгҖҢй–Ӣе§ӢиЁӘе•ҸгҖҚпјҢе…¶д»–иҰҒжҺ’йҡҠгҖӮ

Semaphore зҡ„жҮүз”Ёе ҙжҷҜ

1. API йҷҗжөҒ

дёҚи¶…йҒҺ 10 еҖӢдёҰиЎҢи«ӢжұӮпјҡ

2. иіҮж–ҷеә«йҖЈз·ҡжұ 

еҒҮиЁӯйҖЈз·ҡжұ еҸӘжңү 20 еҖӢйҖЈз·ҡпјҡ

3. и®ҖеҜ«йҺ–пјҲй«ҳзҙҡпјү

Semaphore з”ҡиҮіеҸҜд»ҘеҜҰдҪңи®ҖеҜ«йҺ–зҡ„жҰӮеҝөгҖӮ

volatile й—ңйҚөеӯ—

volatile зўәдҝқе…ұдә«и®Ҡж•ёзҡ„еҸҜиҰӢжҖ§е’ҢжңүеәҸжҖ§пјҢдҪҶдёҚдҝқиӯүеҺҹеӯҗжҖ§гҖӮ

д»ҖйәјжҳҜеҸҜиҰӢжҖ§е•ҸйЎҢ

жІ’жңү volatileпјҢиҝҙеңҲеҸҜиғҪж°ёйҒ и·‘дёҚеҮәдҫҶпјҲе„ҳз®Ўе…¶д»–еҹ·иЎҢз·’ж”№дәҶ flagпјүгҖӮ

з”Ё volatile дҝ®еҫ©

volatile зҡ„дҪңз”Ёпјҡ

  1. жҜҸж¬Ўи®ҖеҸ– - еҫһе…ұдә«иЁҳжҶ¶й«”и®ҖпјҢдёҚеҫһ thread local cache
  2. жҜҸж¬ЎеҜ«е…Ҙ - зӣҙжҺҘеҜ«еҲ°е…ұдә«иЁҳжҶ¶й«”
  3. зҰҒжӯўжҢҮд»ӨйҮҚжҺ’ - зўәдҝқж“ҚдҪңй ҶеәҸ

volatile зҡ„еқ‘пјҡдёҚдҝқиӯүеҺҹеӯҗжҖ§

йӣ–з„¶ volatileпјҢдҪҶ count++ е…¶еҜҰеҲҶжҲҗдёүжӯҘпјҡ

  1. и®ҖеҸ– countпјҲеҒҮиЁӯжҳҜ 10пјү
  2. еҠ  1пјҲи®ҠжҲҗ 11пјү
  3. еҜ«еӣһ count

еӨҡеҖӢеҹ·иЎҢз·’еҗҢжҷӮеҒҡпјҢжңғдёҹеӨұжӣҙж–°пјҡ

дҝ®еҫ©ж–№ејҸпјҡз”Ё AtomicInteger

AtomicInteger дҝқиӯүеҺҹеӯҗжҖ§пјҲдҪҝз”Ё CAS - Compare and SwapпјүгҖӮ

еҜҰеӢҷдҫӢеӯҗ

дҫӢеӯҗ 1пјҡеҹ·иЎҢз·’е®үе…Ёзҡ„еҲқе§ӢеҢ–

йҖҷжҳҜз¶“е…ёзҡ„ double-check lockingпјҲйӣ–з„¶еңЁзҸҫд»Ј Java 8+ жңүжӣҙеҘҪзҡ„еҒҡжі•пјүгҖӮ

дҫӢеӯҗ 2пјҡйҷҗеҲ¶дёҰиЎҢеҹ·иЎҢ

volatile зҡ„жӣҝд»Је“Ғ

зҸҫд»Ј Java жңүжӣҙеҘҪзҡ„йҒёж“Үпјҡ

Atomic* зі»еҲ—жҸҗдҫӣдәҶеҺҹеӯҗжҖ§дҝқиӯүпјҢйӮ„жңү volatile зҡ„еҸҜиҰӢжҖ§гҖӮ

Semaphore vs ReentrantLock vs synchronized

йҒёж“Үпјҡ

  • з°Ўе–®дә’ж–Ҙз”Ё synchronized
  • йңҖиҰҒи¶…жҷӮгҖҒжўқд»¶и®Ҡж•ёз”Ё ReentrantLock
  • йңҖиҰҒйҷҗеҲ¶дёҰиЎҢж•ёйҮҸз”Ё Semaphore

еҜҰеӢҷе»әиӯ°

  1. Semaphore з”Ёж–јйҷҗжөҒе’ҢйҖЈз·ҡжұ 

    • API йҖҹзҺҮйҷҗеҲ¶
    • иіҮж–ҷеә«йҖЈз·ҡжұ 
    • еҹ·иЎҢз·’жұ йҡҠеҲ—з®ЎзҗҶ

  2. volatile з”Ёж–јз°Ўе–®зҡ„зӢҖж…ӢжЁҷиӘҢ

    • еҹ·иЎҢз·’еҒңжӯўж——жЁҷ
    • еҲқе§ӢеҢ–жӘўжҹҘ
    • дёҚиҰҒз”Ёж–јиЁҲж•ёеҷЁ

  3. иЁҲж•ёеҷЁз”Ё AtomicInteger/AtomicLong

    • жҜ” volatile int + жүӢеӢ•йҺ–жӣҙе®үе…Ё
    • жҜ” synchronized жӣҙй«ҳж•Ҳ

  4. иӨҮйӣңзҡ„дёҰиЎҢйӮҸијҜз”Ё concurrency еә«

    • CountDownLatch - зӯүеҫ…еӨҡеҖӢеҹ·иЎҢз·’е®ҢжҲҗ
    • CyclicBarrier - еҗҢжӯҘеұҸйҡң
    • Phaser - еҲҶйҡҺж®өеҗҢжӯҘ
    • ConcurrentHashMap - еҹ·иЎҢз·’е®үе…Ёзҡ„ Map
