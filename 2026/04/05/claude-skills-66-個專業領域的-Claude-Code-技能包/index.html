claude-skills — 66 個專業領域的 Claude Code 技能包

發表於2026-04-05|更新於2026-05-14|Claude Code
|總字數:1.4k|閱讀時間:5分鐘|瀏覽量:

你叫 Claude Code 幫你寫 NestJS 的 JWT 驗證，它給你一段「看起來對但框架慣用法全錯」的程式碼。問它 PostgreSQL 查詢優化，它丟一個教科書等級的答案，但完全沒考慮你的 index 結構。通才的宿命就是這樣——什麼都能聊兩句，但哪個框架都不夠深入。

claude-skills 這個開源專案就是在補這塊缺口。一次安裝 66 個專業領域的 Skill，從 React 到 Kubernetes、從 SQL 優化到 LLM 微調，每個領域都有獨立的深度知識包。裝完之後不需要手動呼叫，AI 會根據你的提問自動觸發對應的 Skill。

Progressive Disclosure——不是一次塞爆 context

AI agent 最怕的事情之一：context window 塞滿。如果 66 個 Skill 的完整內容一次全丟進去，token 直接爆掉，回答品質反而下降。

claude-skills 用了一個叫 Progressive Disclosure（漸進式揭露）的架構。每個 Skill 分兩層：

  • SKILL.md（約 80-100 行）：角色定義、觸發條件、核心流程、reference 路由表
  • **references/**（各 100-600 行）：深度技術參考資料，只在碰到相關問題時才載入

類比一下——SKILL.md 是書的目錄，agent 先看目錄知道有哪些章節，碰到問題時才翻開對應章節。官方說法是可以省約 50% 的 token。實際用起來，確實感受到回應速度有比較快。

安裝方式

四種裝法，看你的習慣：

方式 指令 說明
Marketplace（推薦） /plugin marketplace add jeffallan/claude-skills 完整安裝含 Slash Commands
GitHub 直裝 claude plugin install https://github.com/jeffallan/claude-skills 完整安裝
Agent Skills CLI npx agent-skills-cli@latest add @Jeffallan/claude-skills 支援 42+ AI agent
手動複製 cp -r ./skills/* ~/.claude/skills/ 本地開發用

裝完記得重啟 Claude Code。

自動觸發，不用背 66 個名字

這是最實用的設計——你不需要記住任何 Skill 名稱，直接正常對話就好：

背後機制是每個 SKILL.md 的 description 欄位寫了觸發條件（Use when ...），agent 根據語意匹配自動選擇。觸發不一定 100% 準確，偶爾會漏或誤觸發，但大部分情況蠻穩的。

66 個 Skill 都涵蓋什麼？

快速掃一眼分類：

語言專家（12 個）：Python、TypeScript、JavaScript、Go、Rust、Java、C#、Swift、Kotlin、C++、PHP、SQL。每個都不是泛泛的語法教學，而是進階用法——型別提示、async 模式、效能優化這些。

後端框架（7 個）：NestJS、Django、FastAPI、Spring Boot、Laravel、Rails、.NET Core。重點是遵循 Framework Idiom Principle，參考資料反映該框架的慣用寫法，不是跨框架通用模式。

前端與行動（7 個）：React 19、Next.js App Router、Vue 3、Angular、React Native、Flutter。

基礎設施與 DevOps（10 個）：Kubernetes、Terraform、PostgreSQL、Cloud Architect（AWS/Azure/GCP）、SRE、Chaos Engineer 等等。

API 與架構（6 個）：GraphQL、REST、WebSocket、Microservices、MCP Developer、Architecture Designer。

品質、安全、資料/ML（13 個）：測試、Code Review、文件、安全審查、Pandas、Spark、ML Pipeline、Prompt Engineering、RAG、Fine-Tuning。

特殊領域（11 個）：Salesforce、Shopify、WordPress、Legacy Modernizer、嵌入式系統、遊戲開發，還有一個叫 The Fool 的決策壓力測試工具，用 5 種推理模式（Socratic、Dialectic、Pre-mortem、Red Team、Falsification）挑戰你的技術決策。

附帶 9 個 Workflow Commands

除了 Skills，還附帶 9 個 slash commands 覆蓋專案管理全生命週期——從 Intake（記錄觀察）、Discovery（需求探索）、Planning（建立 Epic/Sprint）到 Execution（執行 Ticket）和 Retrospectives（回顧）。

比較特別的是 Common Ground，它會揭露 Claude 對你專案的隱性假設，支援 --list、--check、--graph 三種模式。這個功能在大型專案裡特別有用，可以避免 AI 默默帶著錯誤假設幫你寫 code。

Workflow Commands 可以整合 Atlassian Jira/Confluence，但需要額外設定 MCP server。

實際使用場景

全端功能開發：Feature Forge 定義需求 → Architecture Designer 設計架構 → 框架 Skill 實作 → Test Master 測試 → Code Reviewer 審查 → DevOps Engineer 部署。一條龍。

安全稽核：Secure Code Guardian 在寫 code 時防堵漏洞，Security Reviewer 做事後滲透測試審查。雙重防線。

遺留系統現代化：Spec Miner 逆向工程現有程式碼為規格，Legacy Modernizer 規劃遷移策略，然後用新框架的 Skill 重新實作。這組合對接手老專案的工程師來說很香。

AI/LLM 應用開發：Prompt Engineer → RAG Architect → Fine-Tuning Expert → MCP Developer，四個 Skill 串起完整鏈路。

要注意的地方

Skills 是靜態知識參考，不是可執行工具或 MCP server。它們教 AI 怎麼做，但不會自己跑起來做事。

版本目前 0.4.11，仍在積極開發中。宣稱支援 11+ 個 AI agent（Claude Code、Cursor、Copilot、Gemini CLI 等），但效果因 agent 而異。另外，即使有 Progressive Disclosure，66 個 Skill 的 SKILL.md 加起來仍有一定的 token 消耗。

自己要寫 Skill 的話，有一個 Description Trap 要注意——description 欄位只寫「做什麼 + 何時用」，絕對不寫流程步驟。不然 agent 會只讀 description 就覺得夠了，跳過完整 SKILL.md 內容。

66 個 Skill 裝完，Claude Code 從通才變成帶著 66 本參考書的通才。不是萬能，但確實讓回答的深度和框架慣用法準確度提升了一個檔次。開源 MIT 授權，值得試試。
