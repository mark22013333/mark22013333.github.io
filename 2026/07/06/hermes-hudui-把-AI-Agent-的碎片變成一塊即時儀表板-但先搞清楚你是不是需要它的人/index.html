hermes-hudui：把 AI Agent 的碎片變成一塊即時儀表板，但先搞清楚你是不是需要它的人

發表於2026-07-06|更新於2026-07-06|AI 開發工具
|總字數:1.5k|閱讀時間:4分鐘|瀏覽量:

先問你一個問題，答對了下面都不用看：你手上那隻 AI agent，現在到底記得多少東西、學會了哪些技能、這個月燒了你幾塊美金——你答得出來嗎？

如果你答得出來，或者你根本不在乎，那這篇可以關掉了。真正會被 hermes-hudui 打中的，是另一種人：agent 已經跑了一陣子，~/.hermes/ 這個資料夾越長越大，裡面塞著 session 紀錄、記憶、技能、排程，散成一地碎片，你隱約覺得該有個地方能一眼看懂它在幹嘛——但又懶得為了這件事去接一套監控平台。

它做的事，比你想的少

大部分監控工具的思路是這樣：讓被監控的對象「主動回報」。裝一個 SDK、埋幾個 telemetry 點、把資料送到某個後台，然後你在儀表板上看。LangSmith、LangFuse 都是這條路，做 trace、做 debugging，很強，但你得先改 agent 的程式碼。

hermes-hudui 反過來。它一行 agent 的程式碼都不碰，只做一件事：盯著 ~/.hermes/ 這個資料夾。agent 把資料寫在哪，它就去讀哪。本質上它不是「AI agent 監控工具」，而是「一個資料夾的即時視覺化工具」——只是這個資料夾剛好裝著一隻 agent 的全部家當。

這個區別很關鍵。因為它不依賴 agent，所以 agent 掛了它還活著，agent 換版本它也不用跟著改。代價是：沒有那個資料夾，它就是一片空白。它不是通用的，它是為 Hermes 這隻 agent 量身訂的。

好處是介面有溫度。它不給你一排冷冰冰的 metrics，而是用「What I Know / What I Remember / What I’m Learning」這種第一人稱的敘事，把 agent 的狀態講給你聽。你看的不是圖表，是一隻 agent 在跟你報告它自己。

一改就動的那條線，其實是三段接力

它號稱即時。檔案一改，畫面就跟著變，中間不用你按重新整理。這件事聽起來很魔法，拆開來看其實就是一條三段接力的線。

想像你家門口裝了個感應器。有人走過，感應器偵測到動作（第一段），透過對講機通知屋裡的你（第二段），你才走到窗邊看是誰（第三段）。hermes-hudui 幹的是一模一樣的事：後端用 watchfiles 這個套件盯著資料夾，檔案一變它就知道（感應器）；知道之後透過 WebSocket 廣播出去（對講機）；前端收到通知，才去重新抓那一塊對應的資料（走到窗邊看）。

聰明的地方在「只看那一塊」。它不是有東西變了就整個畫面重刷，而是靠檔名判斷是哪一類資料變了——state.db 動了，它就知道是 session 要更新，其他 tab 動都不動。前端這邊還有個小心機：只在「第一次載入」顯示 loading，之後每次背景更新，畫面永遠停在上一版資料上無縫換掉，你不會看到那顆討厭的轉圈圈。使用者的體感是——它一直都是對的，從來沒有在「載入中」。

再往深一層，是這個專案最讓我欣賞的部分：它幾乎沒裝什麼東西。後端四個套件，前端三個 runtime 依賴，不需要外部資料庫，不需要額外服務。一個做到這麼多 tab（記憶、技能、session、排程、健康檢查、多 model 花費追蹤）的工具，依賴清單短到你會懷疑是不是漏寫了。這不是省事，這是一種判斷：每多一個依賴，就多一個會壞、要維護、會拖慢安裝的地方。能不加就不加。

但它有幾個地方會咬你

工具再漂亮，判斷該不該用，得看它的邊界在哪。hermes-hudui 有幾條線，你踩到就會不舒服。

最要命的一條：它預設綁在 0.0.0.0，而且沒有任何認證機制。翻成白話——如果你把它跑在一台有公網 IP 的機器上，任何人只要知道網址，就能看光你 agent 的全部資料：記憶、對話、花了多少錢。它從設計上就只準備給你在內網自己看。放到外面去，等於把家門鑰匙插在鎖上還貼了張紙條寫「請進」。

其他幾條沒那麼致命，但你得先知道：它是純唯讀的，只能看不能改，想透過它去調 agent 的設定或清記憶，做不到；前端需要用 Node.js 先 build，不像純 Python 專案可以一句 pip install 打發；那張多 model 的價格表是寫死在原始碼裡的，新 model 上市或降價，你得自己去改；它直接讀 SQLite 的 state.db，萬一 agent 正在瘋狂寫入，偶爾會撞到短暫的 lock。

這些不是 bug，是這類工具的設計取捨——為了「零設定、直接讀檔案」這個爽點，它放棄了「安全、可寫、免 build」。你要嘛接受這組交換，要嘛它不適合你。

所以到底該不該裝

判斷很乾脆，就一條線：你有沒有一隻正在長大的 Hermes agent，而且你會在自己的機器或內網上看它。

有，而且你想搞清楚錢花在哪、agent 這幾天學了什麼、cron 半夜有沒有偷偷跑掛——那它幾乎是現成最順手的選擇，裝起來十分鐘，依賴少到不會拖累你。

沒有那個資料夾，或者你想要的是一個能監控任何 agent、能對外開放、能反向操作的通用平台——那別碰，它不是為你做的，硬套只會處處卡住。工具的價值從來不是功能多寡，是它有沒有為某一種明確的人，把某一件事做到剛好。hermes-hudui 很清楚自己是為誰做的，你也該一樣清楚。
