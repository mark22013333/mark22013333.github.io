CodeBurn — 你該不該裝一個工具盯著 AI Coding 的帳單

發表於2026-06-08|更新於2026-06-08|開發工具
|總字數:1.5k|閱讀時間:4分鐘|瀏覽量:

先問你自己一個問題：你是等帳單來才嚇一跳的人，還是想在嚇到之前就知道數字的人？

這個問題決定了你要不要往下看。因為 CodeBurn 這個工具，對前一種人來說是雞肋，對後一種人來說是早就該裝的東西。它做的事情很單純——讀取你本機上 Claude Code、Cursor、Codex 這些 AI coding 工具存下來的 session 資料，算出你到底花了多少 token、多少錢、花在哪。沒有 API key，沒有 proxy，npx codeburn 一行就能跑。

問題不在它好不好裝，問題在你需不需要。所以這篇不打算逐條念它的功能，而是幫你把「該裝」跟「別裝」的那條線畫清楚。

先講你用不到它的情況

判斷一個工具值不值得，最快的方法是先想它什麼時候是壞選擇。

如果你是 Claude Max 訂閱戶，每個月固定那筆錢就是上限，跑爆了頂多被限速，不會有額外帳單——那 CodeBurn 對你的「省錢」幫助有限。你的錢已經是沉沒成本了。同樣地，如果你一週只開兩三次 AI 工具寫點小東西，花費低到你根本不在意，那裝一個儀表板來盯它，就是用工程的力氣解決一個不存在的問題。

這裡有個更隱性的陷阱：監控本身會上癮。你裝了一個能看即時花費的 menubar app，接下來你可能每寫五分鐘就瞄一眼數字，反而打斷了真正在做的事。油表的存在是為了讓你不用一直擔心沒油，不是讓你每三秒看一次油表。如果你裝它的結果是焦慮變多、產出變少，那它在你身上就是裝反了。

把這兩種情況排除掉，剩下的人才是它真正的受眾。

那哪種人該裝？

最直接的一群，是「按用量付費、而且用得不少」的人。你接 API、你開公司專案、你一天要 AI 改幾十次 code——這種人的花費是浮動的，而浮動的東西不盯著，它就會在某個月給你一個驚喜。CodeBurn 直接讀本機的 JSONL 跟 SQLite，把花費按專案、按模型拆開：哪個專案最燒、哪個模型最貴、cache hit 率健不健康，一目瞭然。

但「看花了多少」其實是它最不值錢的功能。真正有意思的是接下來這幾個。

codeburn optimize 會掃你的 Claude Code 設定，跑十個偵測器找浪費源。最常被抓到的，是那種你早就忘記自己掛著、但每次開 session 都默默載入吃 token 的 MCP server；還有那份越長越肥、塞滿 context window 的 CLAUDE.md。它不只告訴你「這裡有問題」，還給你可以直接複製貼上的修法，最後打一個 A 到 F 的健康分數。這比單純看花費有用太多——因為它把「你花了多少」變成「你哪裡白花了」。

codeburn compare 則是讓兩個模型對決。它會算 Opus 跟 Sonnet 各自的「每次 edit 成本」跟「one-shot rate」——也就是 AI 一次就寫對、不用你來回退它的比率。這個數字很關鍵：很多人憑感覺覺得「貴的模型比較聰明」，但感覺不是數據。也許在你的工作型態下，便宜模型的 one-shot rate 只差一點點，那貴的那幾倍錢就是燒在自我安慰上。

還有一個實驗性的 codeburn yield，把你的 AI session 跟 git commit 對起來，算出你花的 token 最後有多少比例真的 ship 到 main、多少被 revert 掉。這是在問一個更狠的問題：你燒的錢，到底有沒有變成留下來的程式碼。

它刻意不做的事，才是它可信的地方

一個追蹤花費的工具，最諷刺的失敗方式，就是它自己變成另一個花錢、洩漏資料的東西。CodeBurn 在這件事上的設計很克制：純本地、唯讀、確定性。它不呼叫任何 LLM，不把你的 session 往外傳，任務分類完全靠 regex 跟 tool usage pattern 去猜。費用數字是從 LiteLLM 公開的 model pricing 抓的，本地快取 24 小時。

換句話說，它觀測你，但不碰你的東西，也不需要你把資料交出去。這種「最小侵入」的設計，本身就是判斷一個監控工具能不能信任的關鍵——一個要你開 API key、把用量資料上傳雲端才能用的花費追蹤器，光是這個前提就已經自相矛盾了。

當然它有邊界，而且講得很老實。GitHub Copilot 跟 Codex 只追得到 output tokens，費用會低估；Cursor 的 Auto 模式藏了實際用哪個模型，只能用 Sonnet 的價碼去估；optimize 目前也只分析 Claude Code 的 session。這些不是 bug，是「純本地唯讀」這個選擇必然付出的代價——它讀得到的，就是各家工具願意在本機留下的那些。

所以，到底裝不裝

回到最開始那個問題。

如果你的 AI coding 花費是固定的、或低到無感，跳過它，你的注意力有更值得花的地方。如果你的花費是浮動的、而且你已經開始隱約覺得「好像有點貴但說不上來貴在哪」——那個「說不上來」就是你該裝它的訊號。模糊的不安比明確的數字更消耗人，而 CodeBurn 的全部價值，就是把那團模糊換成一張你能看懂、能據以調整的帳。

別把它當成省錢神器，它不會幫你省半毛錢。它只負責讓你看清楚，省不省、怎麼省，是你看清楚之後自己的決定。
