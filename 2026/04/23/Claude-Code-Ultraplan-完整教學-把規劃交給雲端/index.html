Claude Code Ultraplan 完整教學 — 把規劃交給雲端，讓 Claude 替你想 30 分鐘

發表於2026-04-23|更新於2026-05-14|Claude Code
|總字數:1.5k|閱讀時間:4分鐘|瀏覽量:

普通規劃和 Ultraplan 的差距，不是 Claude 有多聰明。是你等不等它想清楚。

在本地跑 plan mode 的時候，Claude 開始思考，你的終端機就被佔住了。你可以等它想，也可以去做別的事，但你不知道它什麼時候想完，也沒辦法在它思考過程中給它回饋。最後它端出一個計劃，你看了之後說「這個方向不對」，然後重來。

Ultraplan 把這個流程整個翻轉。Claude 去雲端想，你繼續在本地工作。計劃生成完，它給你一個連結，你開瀏覽器去 review，可以在特定段落留言、用 emoji 反應表示認可或疑慮、要求局部修改，最後選擇怎麼執行。整個過程 Claude 替你佔用的是雲端算力，不是你的終端機。

它在後台做什麼

輸入 /ultraplan 你的任務描述 之後，會看到一個確認提示。確認之後，流程啟動：

第一步，雲端環境準備。 Claude 在 Anthropic 的雲端伺服器上配置一個臨時容器，把你的 GitHub 倉庫 clone 進去。如果你的 repo 有 setup script，它會執行來安裝依賴、建立環境。

第二步，讀懂 repo。 Claude 在雲端容器裡讀你的 codebase——目錄結構、主要檔案、相關 module。這個步驟讓它對整個專案有完整的 context，不是只靠你的描述。

第三步，起草計劃。 基於對 repo 的理解，Claude 寫出一份完整的實作計劃。這一步可能花 20-30 分鐘，在雲端跑，你的終端機全程可以繼續用。

在瀏覽器裡 review

計劃準備好的時候，終端機會跳出提示和一個瀏覽器連結。計劃呈現在一個專屬的 review 介面裡：

Inline comments：點擊任何段落，留言問問題或提修改意見，Claude 會即時回應並修改對應段落。

Emoji reactions：對一個段落用 emoji 反應（✅ 認可、❓ 疑問、🚩 有問題），不需要打字，Claude 讀到這些 reaction 也會做對應調整。

Outline sidebar：左側有大綱，可以快速跳到計劃的任何一個 section，對於長計劃特別有用。

這個設計讓「規劃 review」變成一個可以異步進行的過程。你不需要在 review 的時候全神貫注地看完整份計劃，可以快速掃過、對有疑問的地方留言，讓 Claude 在你繼續工作的時候在背景修改。

三種執行方式

Review 完成，確認計劃方向沒問題之後，終端機的 dialog 會問你怎麼執行：

Implement here：把計劃注入當前 session 的 context，Claude 從計劃開始繼續執行。適合你已經在這個 session 裡做了一些相關的前置工作。

Start new session：清空當前 session，只帶著計劃開始新的對話。適合需要一個乾淨 context 的長任務，避免之前的對話干擾。

Cancel：把計劃存成檔案，暫時不執行。適合你想先存下計劃、之後決定怎麼用。

使用前的幾個確認

要跑 Ultraplan，三個條件都要滿足：

1. Claude Code v2.1.101 或更新版本。 用 claude --version 確認，版本不夠用 npm install -g @anthropic-ai/claude-code 更新。

2. GitHub-hosted 的 repo。 Ultraplan 需要 clone 你的 repo 到雲端容器，所以 repo 必須在 GitHub 上，且你的帳號有存取權限。本地的 git repo（沒有推到 GitHub）目前不支援。

3. Pro 或 Max 訂閱。 Ultraplan 不在 Free 方案裡，需要 Pro（$20/月）或 Max（$100/月）。

什麼情況下用最划算

Ultraplan 對規劃本身會消耗比較多的 token，不是每個任務都適合用。

值得用的情境：跨多個文件的功能，或涉及架構決策的任務。這類任務需要 Claude 先理解整個 repo 的結構，本地 plan mode 的 context 限制會讓它漏掉重要的關係，Ultraplan 的雲端環境可以讀得更完整。

不划算的情境：單一文件的修改、已經很清楚怎麼做的任務、或 repo 很小的情況。這些用本地 plan mode 或直接執行就夠了。

Ultraplan 的核心價值是：讓你不用在「等 Claude 想清楚」和「做別的事」之間二選一。如果你的任務複雜到需要花 30 分鐘規劃，那個 30 分鐘你本來就不應該坐在終端機前面等著。

跟本地 plan mode 的差別

本地 plan mode Ultraplan
執行位置 本地 雲端容器
終端機是否被佔用 是 否
repo 讀取深度 受 context window 限制 完整 clone
review 方式 在 terminal 看文字 瀏覽器互動介面
需求 任何訂閱 Pro/Max + GitHub repo

這不是「Ultraplan 比較好」的問題，是不同任務選對應的工具。短任務、或你想直接跟 Claude 來回確認方向，用本地 plan 更快。長任務、需要 Claude 深入讀 codebase、你想在規劃階段做異步 review，Ultraplan 才值得用。

接下來可以探索

Ultraplan 和 Claude Code 的其他幾個功能搭配起來效果更好。

搭配 Git Worktrees：用 Ultraplan 先在雲端規劃，再用 Git Worktrees 在本地的多個分支同時執行不同的任務塊，避免不同功能互相干擾。

搭配 /resume：Ultraplan 生成的計劃很長，用 /resume 恢復 session 的時候，v2.1.116 之後的版本對大型 session 的恢復速度快了 67%，不用等太久。

搭配 Monitor tool：執行期間用 Monitor 在背景追蹤進度，不需要一直盯著終端機。

本質上，Ultraplan 是把「規劃」和「執行」切開的機制。切開之後，你可以獨立優化每一段，而不是把它們擠在同一個 session 裡、彼此干擾。
