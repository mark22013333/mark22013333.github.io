AI 與科技新聞摘要 — 2026/03/25

發表於2026-03-25|更新於2026-05-14|AI與科技新聞
|總字數:1.6k|閱讀時間:5分鐘|瀏覽量:

三月的最後一週，AI 的版圖在跳舞。Anthropic 把電腦控制權交給 Claude，結果反而跟五角大廈打起官司；OpenAI 花錢買了個 Python 工具鏈，要讓 AI 不只生代碼，還能管工程流程；而 MCP 生態繁花盛開，一邊是新工具層出不窮，一邊是人人都在忙著堵安全漏洞。Google 速度比誰都快，Gemini 3.1 Flash-Lite 便宜到不行，Claude 的用量限制還暫時加倍——三天內把能用的額度全燒了，再見了各位。

Anthropic Claude 電腦控制正式上線——AI 手伸進你的電腦了

這週 Anthropic 最大的新聞：Claude 可以從手機發指令，然後 AI 會自己開應用程式、操作瀏覽器、填寫試算表。不只是生成程式碼那麼簡單，是真的在你電腦上幫你點按鈕。

auto mode 的安全機制怎麼做的？ 每個動作執行前都要過一道審查，系統會檢查有沒有風險行為、有沒有 prompt injection 企圖。聽起來防護還算周密，但新功能上線第一天就被拿去測試安全漏洞，這很正常——真實世界永遠比實驗室複雜。

限制條件也蠻嚴的：只有 Claude Pro 和 Claude Max 訂閱者能用，而且目前只在 macOS 上支援。Windows 和 Linux 還在排隊。

這有夠猛的一件事是，Anthropic 同時在法庭上為 Claude 能用在什麼地方跟五角大廈吵架。聯邦法官直接嘲諷五角大廈，說把 Anthropic 列為「供應鏈風險」的做法「看起來像是在試圖癱瘓 Anthropic」。Anthropic 的主張很硬：Claude 不能拿來做全自動武器或大規模監控美國人。大科技公司跟政府部門的較勁，永遠都這麼刺激。

原文來源：Anthropic hands Claude Code more control, but keeps it on a leash - TechCrunch、Anthropic’s Claude AI agent use computer to finish tasks - CNBC、Pentagon Anthropic hearing, judge ‘troubling’ - CBS News

OpenAI 買下 Astral，要把 Python 工具生態塞進 AI Codex

OpenAI 上週宣布收購 Astral，這家新創開發的 uv、Ruff、ty 等工具在 Python 圈超熱。uv 是個超快的套件管理器，Ruff 是 linter + formatter，ty 做型別檢查——基本上覆蓋了 Python 開發者的日常痛點。

為什麼 OpenAI 要買他們？很簡單，因為現在的 AI coding 還停留在「幫你寫幾行函數」的層級。真實開發需要套件管理、code style 統一、相依性解析，整個工程流程。Astral 的工具會整進 Codex（OpenAI 的 AI 編碼代理），讓模型不只會寫代碼，還能參與整個開發工作流——從建專案、安裝依賴、檢查格式，到交付都包。

這招有點狡猾。OpenAI 知道光有模型能力還不夠，開發者真正要的是一個能融入他們現有工具鏈的東西。買個已經被 Python 社群採納的工具，比從零開始打造快多了。

原文來源：OpenAI to acquire developer tooling startup Astral - CNBC

MCP 生態 2026 路線圖：安全、治理、企業就緒缺一不可

Model Context Protocol 這週發布了 2026 年的方向圖：transport 擴展、Agent 間通訊、治理成熟化、企業級就緒。聽起來平凡，但背後代表著 MCP 在往「不只是玩具協定」的方向走。

同步推出的工具也在撐這個方向：SurePath AI 發布了 MCP Policy Controls，可以即時控管 MCP server 和工具的使用——簡單說，就是讓你指定 LLM 能呼叫哪些工具，不能做什麼。Fingerprint 弄了個開源的 MCP Server 防詐欺工具，Google 甚至發布了 Universal Commerce Protocol (UCP)，直接相容 MCP。

有趣的是，安全和治理成了路線圖的核心。上週才爆出一堆 MCP 安全漏洞，這週 Anthropic 他們就急著補進協定設計裡。從「誰都能接」變成「有門檻的接取」，這算是負責任的進化吧。

原文來源：2026 MCP Roadmap - Model Context Protocol Blog、Model Context Protocol Roadmap 2026 - The New Stack

Google Gemini 3.1 Flash-Lite 一出，「便宜好用」成了新標配

Google 推出新的效率模型 Gemini 3.1 Flash-Lite，數字會說話：比前代快 2.5 倍、輸出快 45%、價格卻只要 $0.25/M input tokens。真香。

這代表什麼？一年前還在拚「誰的模型更強」，現在已經進入「怎麼做得比對手便宜又快」的模式。Flash-Lite 的目標群就是那些對速度敏感、成本也敏感的應用——做客服 chatbot、文件處理、批量任務都合適。

Google 在成本戰上氣勢兇猛。當 OpenAI 和 Anthropic 的高端模型價格停留在原地，Google 就靠著效率版本一路降價。開發者挑選模型的時候，便宜和快真的會輸掉其他因素。

原文來源：Google Gemini 3.1 Flash-Lite announcement

Claude 用量限制臨時加倍——離峰免費吃到飽，但只到 3/28

Anthropic 在離峰時段臨時把用量限制加倍到 3 月 28 日。Free、Pro、Max、Team 訂閱者全部受惠。這看起來像是為了平衡伺服器負載的臨時措施，或者是某種「我們有多餘運算量」的宣傳。

但從使用者角度看，這三天多的額度加倍期間，很多人直接把 prompt 的長度、請求次數都調高了。真到 3/29 回歸正常限制時，才開始後悔——太爽了收手停不下來。

原文來源：AI usage limits are becoming the new reality for consumers - PYMNTS

今日觀察

電腦控制、代碼工具整合、協定安全化、模型持續降價——這幾條線串起來就是現在 AI 市場的樣子。

Anthropic 和 OpenAI 都意識到，光讓模型變強不夠，得讓模型能進到開發者的工作流裡才行。一個靠電腦控制權，一個靠收購工具鏈，招數不一樣，目標一樣：成為開發者每天都要用的基礎設施。

MCP 生態繁榮的反面是安全得每週打補丁，這也透露出一個現實——協定越開放越靠近系統層，安全隱患就越多。幸好各家都在認真對待，不是放任不管。

Google 的 Flash-Lite 則代表另一種競爭邏輯——當頂級性能已經夠用，比拚的就回到基本面：便宜、快、可靠。三個都佔的產品，在市場上就很難輸。至於 Claude 用量限制加倍這三天，多數開發者大概都會留著美好回憶，然後默默調低請求頻率。
