AI 與科技新聞摘要 - 2026/04/15

發表於2026-04-15|更新於2026-05-14|AI與科技新聞

用戶發現 Claude 變笨了，Fortune 一篇報導把 Anthropic 推上風口浪尖。同一時間，AI coding 工具的軍備競賽繼續加速：Kiro CLI 2.0 進 headless 和 Windows、Gemini CLI 玩出 Chapters 新概念、OpenAI Codex 的 MCP 整合終於像樣了、Claude Cowork 悄悄 GA。週三早上的重點整理。

Anthropic 被抓包偷降 Claude 運算量 - 用戶信任危機

Fortune 昨天（4/14）發了一篇長文，標題很直接：Anthropic 正面臨用戶反彈潮。

事情的起因是越來越多重度用戶反映 Claude 的表現變差了——不遵守指令、走捷徑、複雜任務的錯誤率明顯上升。社群開始挖原因，有人丟出了一個數據聲稱效能下降 67%，雖然方法論有爭議，但引爆了大量共鳴。

VentureBeat 跟進報導揭露了核心問題：Anthropic 悄悄調低了 Claude 的預設「effort」層級來節省 token 消耗。翻譯成白話就是——為了省運算成本，讓模型預設用比較少的算力回應。Anthropic 的回應是承認有做這個調整，但強調這是「平衡 token 使用」而非「偷偷降低品質」。

問題在於「沒說就改」這件事本身。Anthropic 比任何 AI 公司都更強調透明度，這次的處理方式直接打臉自己的品牌定位。Fortune 文章指出，這件事發生的時機很敏感——Anthropic 正在準備可能的 IPO，用戶成長飛速，但大型資料中心合約的規模卻比競爭對手少。外界開始猜測 Anthropic 是不是算力不夠了。

不管真實原因是什麼，這波信任損傷是實打實的。對開發者來說，模型行為的可預測性比跑分還重要。你可以接受模型不夠強，但不能接受它昨天還好好的今天突然變了。

Kiro CLI 2.0 - Headless 模式和 Windows 支援

AWS 的 Kiro CLI 發布了 2.0 版本，三個重點更新。

Headless 模式是最實用的：現在可以不開瀏覽器跑 Kiro CLI，直接塞進 CI/CD pipeline、容器建置、自動化工作流。對需要在伺服器端跑 AI coding agent 的團隊來說，這是基本功能但之前一直缺。

Windows 支援終於來了。Kiro CLI 現在可以在 Windows Terminal 裡面跑，不用再靠 WSL 繞路。搭配上個月的 Kiro IDE 正式發布，AWS 在 AI coding 工具這塊的產品線已經相當完整。

子代理監控也是新功能——在實驗性的 Terminal UI 裡面可以看到 sub-agent 的執行進度，還有更新過的 todo list 和 task list。跟 Claude Code 的 sub-agent 體驗類似，但介面整合度更高。

Kiro CLI 2.0 搭配免費的 Kiro IDE，在 spec-driven development 這個定位上越來越有自己的調性。不像 Cursor 追求「對話式 coding」，Kiro 走的是「先寫規格再寫碼」的路線。

原文來源：Introducing enhancements in Kiro CLI 2.0

Gemini CLI v0.37.2 - Chapters 讓長對話終於有結構了

Google 的 Gemini CLI 在 4/13 發布了 v0.37.2，最亮眼的新功能叫 Chapters。

Chapters 的概念是根據工具使用和意圖，自動把 agent 的互動過程分成邏輯章節。跑一個複雜任務的時候，你不再看到一長串混在一起的 log，而是被分成「分析需求」「修改檔案」「跑測試」這樣的段落。長 session 的可讀性直接提升一個層級。

另一個重要更新是 Dynamic Sandbox Expansion——Linux 和 Windows 上都支援動態擴展沙盒和 worktree，在受限環境裡的開發彈性更好了。瀏覽器 agent 也升級了，加入持久 session 管理和動態 read-only 工具發現。

回頭看 4/1 發的 v0.36.0 也有值得注意的東西：multi-registry 架構、macOS Seatbelt 和 Windows 原生沙盒。這些都是在安全性上的投資，跟 Claude Code 的 sandbox 策略走類似方向。

CLI 工具的競爭已經從「誰的模型比較強」轉向「誰的工程體驗比較好」。Chapters 是一個很聰明的切入點——不改模型能力，只改資訊呈現方式，但使用體驗差很多。

原文來源：Gemini CLI v0.37.2 Release Notes

OpenAI Codex MCP 整合大升級

OpenAI Codex 這一輪更新的重點在 MCP 整合。

MCP Apps 和自訂 MCP server 現在支援 resource reads、工具呼叫 metadata、自訂 server 的工具搜尋、server-driven elicitations、檔案參數上傳，還有更穩定的 plugin cache 刷新。這些聽起來零碎，但加在一起代表 Codex 的 MCP 生態系終於從「能用」變成「好用」。

Remote 工作流也有大更新：egress websocket transport、remote --cd 轉發、runtime 遠端控制啟用、sandbox-aware 檔案系統 API，還有實驗性的 codex exec-server 子指令。

計費方面有個對企業用戶友善的變化：ChatGPT Business 和 Enterprise 團隊現在可以加 Codex-only 席位，pay-as-you-go 計費，不用綁固定月費。對只需要 coding agent 不需要完整 ChatGPT 的團隊來說，這省不少錢。

TUI 也有小更新：Ctrl+O 複製最新 agent 回應、/resume 可以直接跳到指定 session。

原文來源：Codex Changelog

Claude Cowork 正式 GA - macOS 和 Windows 都能用了

Claude Cowork 正式脫離 preview 階段，在 macOS 和 Windows 的 Claude Desktop app 中正式 GA。

GA 版本加入了 expanded analytics、OpenTelemetry 支援，以及 Enterprise plan 的 role-based access controls。對企業來說最重要的是最後一項——終於可以控制誰能用 Cowork、用到什麼程度。

同時 Claude Desktop 的 Excel 和 PowerPoint add-ins 也做了升級，可以共享完整對話 context。在 Excel 裡做的事情，PowerPoint 那邊也知道。跨應用的 context 共享是 Cowork 相比其他 AI 工具的差異化賣點之一。

Claude 的 inline visualizations 功能也正式上線——可以在回應中直接產生圖表和 diagram，不用再另外開 artifacts。

Cowork GA 的時間點有點微妙，剛好碰上用戶反彈潮。產品在擴張，但底層模型的信任正在被質疑。Anthropic 接下來怎麼處理這個落差，比任何功能更新都重要。
